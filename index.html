What is CI/CD?

Continuous Integration (CI) and Continuous Deployment (CD) form a pipeline that automates the software delivery process. CI automatically tests and integrates code changes into a shared repository, ensuring early bug detection. CD automatically deploys the integrated code to production environments like web servers or cloud platforms. Together, they eliminate manual deployment tasks, reduce human error, and enable faster, more reliable updates.

My Pipeline

When I push new code to the 'main' branch on GitHub, a GitHub Actions workflow is triggered automatically. This workflow checks if my essential files exist (CI), packages them, and then seamlessly deploys the updated files to GitHub Pages (CD).
